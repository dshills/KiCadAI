# KiCad-Backed Schematic Anchor Calibration

## Goal

Make direct schematic wires generated by KiCadAI attach to the physical KiCad
pins of embedded symbols at rotations `0`, `90`, `180`, and `270`, with each
supported X/Y mirror state. This is a required correctness layer for
human-readable layouts: a short local wire may only be emitted when its
endpoints are provably real electrical anchors.

## Scope

This work covers the schematic writer, `schematicir` layout placement model,
embedded symbol templates, and KiCad-backed tests. It applies to generated
schematics only. The IR gains an explicit constrained mirror field because
arbitrary readable schematic layout cannot safely omit a normal KiCad symbol
orientation state.

The first calibration corpus covers these representative built-in families:

- `Device:R` and `Device:C`: vertical passive pins;
- `Connector_Generic:Conn_01x02`: asymmetric multi-pin connector geometry;
- `Amplifier_Operational:LMV321`: mixed horizontal, vertical, input, output,
  and power pins.

## Non-Goals

- PCB placement, routing, or DRC changes;
- new circuit blocks or component selection;
- changing AI intent or IR schema semantics;
- silently converting explicit `use_label: false` requests to labels;
- claiming every installed KiCad library is calibrated in this milestone.

## Problem

KiCad symbol-library coordinates invert Y when parsed into `SCH_PIN` local
positions. Its dangling-endpoint pass, however, obtains an embedded symbol's
wire attachment point through `SCH_SYMBOL::GetPinPhysicalPosition` on the
library pin. Generated embedded bodies must therefore retain the serialized
library-pin endpoint convention used by that pass; the separate anchor
metadata used by layout and transaction generation must use the same
convention. Mixing parser-local and library-pin coordinates can produce wires
that are orthogonal and internally consistent but do not touch KiCad's actual
connection endpoint after symbol rotation.

KiCad's `SCH_IO_KICAD_SEXPR_PARSER::parseXY(true)` and
`SCH_SYMBOL::GetPinPhysicalPosition` are the source-of-truth implementation
for this contract. KiCadAI must use one explicitly named schematic-space
anchor representation for placement, routing, transaction emission, label
stubs, and post-write validation.

## Required Behavior

1. Template pin anchors represent physical schematic-space offsets relative to
   the symbol origin, not raw library-file coordinates.
2. A calibrated anchor transformed by `0`, `90`, `180`, or `270` degrees and
   each supported mirror state must agree with KiCad's physical pin location.
3. `schematiclayout`, `schematicir`, and `designapi.Builder` must consume the
   same connection-anchor value.
4. Direct paths must begin and end on those anchors and remain orthogonal.
5. KiCad ERC must report no `wire_dangling`, `pin_not_connected`, or
   connection errors for each calibration fixture.
6. Explicit `use_label: false` remains direct-only. If a route cannot be
   emitted with proven anchors or clean readability, generation must fail with
   an actionable issue rather than silently emitting label connections.
7. Automatic layouts may continue to use the existing deterministic label
   fallback for long, high-fanout, crowded, or otherwise unsuitable nets.
8. All anchors and wire endpoints must lie on KiCad's 1.27 mm connection grid.
9. The generated schematic must round-trip through KiCad with no normalized
   diff.
10. Block-generated `PinSpec` metadata, label anchors, no-connect markers, and
    electrical-preflight wires must use the same calibrated physical anchors as
    the writer. Preflight doglegs must avoid unrelated pin and no-connect
    anchors rather than treating a geometrically clean crossing as electrical
    evidence.

## Anchor Representation

`TemplatePin.Offset`, resolver `SymbolPin.Position`, and transaction
`PinSpec` offsets are schematic-space coordinates. Raw `lib_symbols` source
text remains library-file syntax and is never reused as a connection offset
without the KiCad coordinate conversion.

Connection overrides are allowed only where KiCad-backed evidence proves a
template's physical anchor differs from the default resolved anchor. Each
override must identify its reason and have a focused regression test.

## Calibration Corpus

Each fixture contains a driven/direct net with `use_label: false`, all unused
pins marked no-connect where necessary, and a small unambiguous layout. The
corpus runs each orientation/mirror combination that the IR supports and must
include:

- a rotated resistor connected to a driven source and receiver;
- a rotated capacitor connected to a driven source and receiver;
- both pins of a rotated connector;
- every LMV321 signal/power pin required for a powered, electrically valid
  op-amp stage.

For each symbol family, the four rotations are separate deterministic cases.
The test output must retain enough artifact data to report the symbol, pin,
rotation, and emitted wire endpoint when KiCad rejects a connection.

## Calibrated Inventory

- `Device:R`: calibrated for all four rotations and both supported mirror
  axes. The KiCad-backed direct-wire matrix requires zero ERC findings and a
  normalized zero-diff round trip for all twelve transform combinations.
- `Device:C`: not calibrated. Its second pin currently fails the isolated
  KiCad direct-wire probe despite matching the visible library geometry. It
  must remain on raw-template/label-safe behavior until a separate empirical
  endpoint contract is proven.
- `Sensor:Generic_I2C`: pins 1-4 use verified local-library physical anchors;
  generated I2C block metadata uses those anchors for wires, labels, and
  no-connect evidence. Pin 5 retains its direct right-side anchor.

## Validation

Every calibration case must run:

1. strict IR decode and transaction validation;
2. strict schematic readability validation;
3. internal schematic electrical validation;
4. project-scoped KiCad CLI ERC with `--severity-all` and zero findings;
5. KiCad save/round-trip normalized equality.

The KiCad lane remains environment-gated, but it is mandatory for promotion of
a calibrated template family. Each test runs in its own temporary project and
uses the checked-in generated ERC settings/project defaults; the command line
always includes `--severity-all`. Unit tests must cover coordinate conversion
and assert that all generation layers use the same anchor helper. The harness
must additionally compare the emitted endpoints against KiCad's parsed symbol
pin positions, rather than treating a clean ERC result alone as proof.

## Rollback

Template anchor changes can move existing generated wires. Make each family a
separate commit, preserve existing promoted fixtures, and revert any family
change that regresses their readability, ERC, or round-trip evidence. Store a
checked-in calibrated-anchor inventory and compare fixture output before and
after each family change so an intentional migration is visible in review.

## Completion Criteria

This calibration project is complete when the four representative families
pass the complete rotation corpus with direct wires, existing generated
schematic fixtures remain clean, and the arbitrary-layout documentation states
the calibrated versus resolver-required boundary precisely. The current
resistor result is a completed family milestone, not completion of the full
calibration project.
